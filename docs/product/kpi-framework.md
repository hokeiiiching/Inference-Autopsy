# KPI Framework

## Purpose

Define product and quality metrics for Inference Autopsy itself.

## Reference When

Use when evaluating releases, roadmap choices, and product quality.

## AI Agents Must Obey

Do not optimize vanity metrics over user trust and reproducibility.

## Product KPIs

- Time to first useful benchmark.
- Percentage of runs with complete methodology metadata.
- Percentage of failed runs with actionable errors.
- Report shareability: single artifact opens without server.
- CI adoption: diff gates usable in common workflows.

## Quality KPIs

- Trace schema compatibility.
- Test coverage for metrics and gates.
- Provider compatibility fixtures.
- Report generation success rate.
- Default test suite runtime.

## Diagnostic KPIs

- Diagnosis backed by measurable evidence.
- False confidence avoided by uncertainty language.
- Next tests included when diagnosis is ambiguous.
