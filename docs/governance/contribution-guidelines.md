# Contribution Guidelines

## Purpose

Define expectations for contributors.

## Reference When

Use before opening issues, PRs, or discussions.

## AI Agents Must Obey

Make contributions easy to review and aligned with project scope.

## Contributor Expectations

- Read `README.md` and `docs/README.md`.
- Keep PRs focused.
- Include tests for behavior changes.
- Update docs for public behavior changes.
- Avoid unrelated formatting churn.
- Explain tradeoffs in PR descriptions.

## Good First Contributions

- Fixture traces.
- Provider compatibility tests.
- Documentation improvements.
- Small metrics edge cases.
- Report accessibility improvements.

```md
## Summary

## Verification

## Docs Updated

## Risks
```
