# Accessibility

## Purpose

Ensure reports are usable by keyboard users, screen readers, and users with low
vision.

## Reference When

Use when creating report templates, tables, charts, forms, or interactive
filters.

## AI Agents Must Obey

Accessibility is part of done, not a polish pass.

## Rules

- Use semantic HTML.
- Provide table headers and captions.
- Do not rely on color alone.
- Maintain WCAG AA contrast.
- Make interactive controls keyboard reachable.
- Provide visible focus states.
- Add chart summaries or data tables for important charts.
- Use clear page titles and section headings.

## Chart Accessibility

Every critical chart should include:

- A concise text takeaway.
- Units.
- Data table or source values.
- Clear legend.

## Report States

Empty and error states must explain what data is missing and how to produce it.
