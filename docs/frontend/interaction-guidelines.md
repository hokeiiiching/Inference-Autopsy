# Interaction Guidelines

## Purpose

Define interactions for reports and future local interfaces.

## Reference When

Use when adding filters, tabs, expandable sections, sorting, or chart controls.

## AI Agents Must Obey

Interactions must clarify data, not hide it.

## Rules

- Use tabs for switching related views.
- Use toggles for binary display options.
- Use menus for option sets.
- Use sliders or inputs for numeric thresholds.
- Use icon buttons with labels or tooltips for common actions.
- Preserve key summary data without interaction.

## Feedback

- Show selected filters.
- Show empty results when filters remove all rows.
- Provide reset controls.
- Keep interactions keyboard accessible.

## Latency Perception

For generated reports, avoid loading spinners when static content can be
rendered immediately.
