# Responsive Design

## Purpose

Define responsive behavior for reports.

## Reference When

Use when changing report layout, tables, charts, or navigation.

## AI Agents Must Obey

Reports must remain readable on laptops, wide monitors, tablets, and phones.

## Breakpoints

- Small: below 640px.
- Medium: 640px to 1024px.
- Large: above 1024px.

## Rules

- Use single-column layout on small screens.
- Keep metric tables horizontally scrollable when needed.
- Preserve units and labels when columns collapse.
- Avoid overlapping text.
- Give charts stable minimum heights.
- Do not hide methodology on mobile.

## Table Rules

- Use tabular numbers.
- Keep row labels visible.
- Avoid truncating metric names.
- Provide compact comparison views for mobile.
