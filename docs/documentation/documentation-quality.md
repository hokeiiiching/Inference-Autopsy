# Documentation Quality

## Purpose

Define standards for all repository documentation.

## Reference When

Use when writing or reviewing docs.

## AI Agents Must Obey

Docs must be current, concrete, and useful for execution.

## Rules

- Start with purpose.
- Say when to reference the file.
- Say what AI agents must obey.
- Prefer examples and templates.
- Keep public contracts synchronized with code.
- Use plain language.
- Include units in metric docs.
- Avoid duplicating long policy across many files.
- Link to canonical docs when possible.

## Freshness

Update docs in the same PR as behavior changes. If a doc is intentionally
deferred, create a tracked follow-up with owner and reason.

## Ownership Metadata

For substantial docs, include owner or area when the project has maintainers
assigned.
