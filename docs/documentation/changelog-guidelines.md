# Changelog Guidelines

## Purpose

Define changelog expectations.

## Reference When

Use before releases and user-visible changes.

## AI Agents Must Obey

User-visible changes need changelog entries.

## Format

Use categories:

- Added.
- Changed.
- Fixed.
- Deprecated.
- Removed.
- Security.

## Entry Rules

- Write for users, not only maintainers.
- Mention CLI, trace schema, or report changes explicitly.
- Include migration notes for breaking changes.
- Link to ADRs or RFCs for major decisions.
