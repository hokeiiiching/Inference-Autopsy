# Technical Debt

## Purpose

Track intentional compromises and future cleanup.

## Reference When

Use before refactoring, planning, or adding related features.

## AI Agents Must Obey

Do not create new debt without documenting owner, reason, and review trigger.

## Debt Template

```md
## <debt item>

Status: active | paid | superseded
Owner: <name>
Created: YYYY-MM-DD
Review trigger: <date, milestone, or symptom>

Context:
<why this compromise exists>

Risk:
<what could go wrong>

Paydown Plan:
<smallest cleanup path>
```
